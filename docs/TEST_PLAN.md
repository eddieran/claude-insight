## Test Framework
- **Unit + Integration:** Vitest

## Test Fixtures Strategy
Real data fixtures from existing Claude Code sessions:
- Hook payloads from `.research/claude-hook-probe/out/*.json` (all 5 event types captured)
- Transcript JSONL snippets from `~/.claude/projects/` (sanitized, covering 20+ entry types)
- Settings.json samples from `~/.claude/settings.json` (sanitized)
- Generated fixtures for edge cases (malformed JSON, huge payloads, unicode)

## Layer 3: Real E2E Tests (Claude Code + claude-insight)

These tests launch actual `claude -p` sessions and verify claude-insight captures everything correctly.

### Test Harness

```
test/e2e/harness.ts:
  1. Start claude-insight daemon
  2. Run `claude-insight init --global` (installs hooks)
  3. Execute a scripted `claude -p` session
  4. Wait for events to propagate (poll daemon health endpoint)
  5. Query claude-insight API/SQLite
  6. Assert captured data matches expected evidence graph
  7. Cleanup: remove hooks, stop daemon, delete test DB
```

### E2E Test Scenarios

| # | Test | Claude -p Command | What to Verify |
|---|------|-------------------|----------------|
| E1 | Simple prompt | `echo "what is 2+2" \| claude -p` | SessionStart captured, UserPromptSubmit captured with prompt text, Stop captured with last_assistant_message |
| E2 | Tool usage | `echo "read the file package.json" \| claude -p` | PreToolUse for Read tool, PostToolUse with file content, tool_invocations row with correct tool_name + tool_use_id |
| E3 | Bash execution | `echo "run ls -la" \| claude -p` | PreToolUse for Bash, PostToolUse with stdout, permission_decisions row (if permission mode requires approval) |
| E4 | File edit | `echo "create a file called test.txt with hello world" \| claude -p` | PreToolUse for Write, PostToolUse success, FileChanged event if watched |
| E5 | Multi-tool session | `echo "read package.json then create summary.md" \| claude -p` | Multiple tool_invocations, correct ordering, parent promptId links all tools |
| E6 | Permission denial | `echo "delete all files" \| claude -p --permission-mode dontAsk` | PermissionDenied event captured with reason |
| E7 | Instruction loading | Run claude -p in a directory with CLAUDE.md | InstructionsLoaded event with file_path, memory_type, load_reason |
| E8 | Transcript correlation | Any session | Transcript JSONL entries correlate with hook events by session_id + tool_use_id |
| E9 | Session metadata | Any session | sessions table has correct cwd, model, claude_version, permission_mode |
| E10 | Cost tracking | Any session with tool usage | sessions.cost_usd populated (from Stop event or transcript) |
| E11 | Subagent | `echo "use an agent to search for TODO comments" \| claude -p` | SubagentStart/Stop events, subagent transcript tailed and linked |
| E12 | Long session | Send 20 sequential prompts | All 20 prompts captured, event ordering preserved, no dropped events |
| E13 | Daemon crash recovery | Kill daemon mid-session, verify JSONL fallback | Events written to backlog.jsonl, processed when daemon restarts |

### E2E Verification Assertions

For each E2E test, verify:
1. **Completeness:** Every hook event that fired has a raw_events row
2. **Correlation:** tool_use_id joins between hooks and transcript entries are correct
3. **Ordering:** Events are stored in chronological order per session
4. **Content integrity:** Payload JSON is valid and matches what Claude Code sent
5. **Normalization:** Typed tables (sessions, tool_invocations, etc.) are populated correctly
6. **FTS:** Ingested events are searchable via FTS

## Test Data Management

### Fixture Generation Script
```
test/fixtures/generate.ts:
  1. Copy sanitized hook payloads from .research/claude-hook-probe/out/
  2. Copy sanitized transcript snippets from ~/.claude/projects/ (first 100 lines of 3 sessions)
  3. Generate synthetic fixtures for edge cases:
     - Malformed JSON (truncated, invalid encoding)
     - Extremely long payloads (>1MB tool response)
     - Unicode edge cases (emoji, CJK, RTL, zero-width chars)
     - All 27 hook event types with minimal valid payloads
     - All 20+ transcript entry types
  4. Write to test/fixtures/
```

### Test Database Seeding
```
test/helpers/seed-db.ts:
  1. Create in-memory SQLite database
  2. Run schema creation
  3. Insert fixture data across all tables
  4. Return seeded DB handle for test use
```
